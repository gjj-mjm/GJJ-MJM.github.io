# Git本地仓库设置与上传步骤指南

## 前言

由于PowerShell执行策略限制，我们将使用更简单的方法来设置Git环境并初始化本地仓库。请按照以下步骤操作：

## 步骤1：检查Git是否已正确安装

1. 打开命令提示符（CMD）：
   - 按下 `Win + R` 键
   - 输入 `cmd` 并按回车

2. 在命令提示符中输入：
   ```
   git --version
   ```

3. 如果显示Git版本信息，则Git已安装；如果提示"不是内部或外部命令"，请先安装Git。

## 步骤2：安装Git（如果尚未安装）

1. 访问官网下载Git安装包：[https://git-scm.com/download/win](https://git-scm.com/download/win)
2. 运行安装程序，使用默认选项即可
3. 安装完成后，重新打开命令提示符，输入`git --version`验证安装

## 步骤3：设置Git用户信息

在命令提示符中执行以下命令（替换为您自己的信息）：

```
git config --global user.name "您的用户名"
git config --global user.email "您的邮箱"
```

## 步骤4：初始化本地仓库

1. 在命令提示符中切换到项目目录：
   ```
   cd c:\trae
   ```

2. 初始化Git仓库：
   ```
   git init
   ```

## 步骤5：添加文件到暂存区

执行以下命令将所有文件添加到暂存区：

```
git add .
```

## 步骤6：提交文件到本地仓库

执行提交命令：

```
git commit -m "初始化项目 - 乡村人才小程序项目"
```

## 步骤7：验证提交是否成功

查看提交历史：

```
git log
```

## 常见问题解决

1. **问题**：提示"git 不是内部或外部命令"  
   **解决**：检查Git是否正确安装，以及是否添加到系统环境变量

2. **问题**：执行脚本文件时提示"无法识别"  
   **解决**：使用命令提示符(CMD)而不是PowerShell执行命令

3. **问题**：添加文件时出现错误  
   **解决**：确保`.gitignore`文件正确配置，避免添加不必要的文件

## 后续操作

如果您需要将本地仓库推送到远程仓库（如GitHub、GitLab等），请先在远程平台创建仓库，然后在本地执行：

```
git remote add origin https://远程仓库地址.git
git push -u origin master
```

---

按照以上步骤操作，您就可以成功设置Git环境并将项目上传到本地仓库了！
