Git安装指南 - Windows系统

注意：本指南适用于Windows系统安装Git。安装完成后，您将能够使用Git命令行工具将文件上传到GitHub仓库。

第1步：下载Git安装包

访问Git官方网站下载最新版本的Git安装包：

下载Git for Windows

根据您的Windows系统位数（32位或64位）选择相应的安装包。如果不确定，可以按以下步骤查看：

  • 右键点击"此电脑"或"计算机"
  • 选择"属性"
  • 查看"系统类型"信息

第2步：安装Git

  1. 双击下载的安装包（通常是一个.exe文件）
  2. 在安装向导中点击"Next"
  3. 选择安装位置，建议使用默认位置，然后点击"Next"
  4. 选择组件，保留默认选项，点击"Next"
  5. 选择开始菜单文件夹，保留默认选项，点击"Next"
  6. 选择Git默认编辑器：
    • 建议选择您熟悉的编辑器，如Notepad++或Visual Studio Code
    • 如果没有特别偏好，可以选择默认的Vim
    选择后点击"Next"
  7. 调整PATH环境变量：
    • 建议选择第二个选项："Git from the command line and also from 3rd-party software"
    • 这将允许在命令提示符和其他软件中使用Git
    选择后点击"Next"
  8. 选择SSH客户端，保留默认选项，点击"Next"
  9. 选择HTTPS传输后端，保留默认选项，点击"Next"
  10. 配置行尾符号转换：
    • 建议选择第一个选项："Checkout Windows-style, commit Unix-style line endings"
    选择后点击"Next"
  11. 配置终端模拟器，建议选择"Windows Terminal"或默认选项，点击"Next"
  12. 配置额外选项，保留默认选项，点击"Next"
  13. 等待安装完成，然后点击"Finish"

第3步：验证Git安装

  1. 打开命令提示符（CMD）或PowerShell
  2. 输入以下命令检查Git版本：
    3. git --version
  3. 如果安装成功，您将看到Git的版本信息，例如：
    4. git version 2.40.1.windows.1

第4步：配置Git用户信息

在使用Git之前，您需要配置您的用户名和邮箱地址。这些信息将用于您的Git提交记录中。

  1. 打开命令提示符（CMD）或PowerShell
  2. 输入以下命令配置用户名（将"Your Name"替换为您的GitHub用户名）：
    3. git config --global user.name "Your Name"
  3. 输入以下命令配置邮箱地址（将"your.email@example.com"替换为您的GitHub注册邮箱）：
    4. git config --global user.email "your.email@example.com"
  4. 验证配置是否成功：
    5. git config --list

    您应该能看到您配置的用户名和邮箱地址。

第5步：配置SSH密钥（可选但推荐）

配置SSH密钥可以让您无需每次推送代码时都输入GitHub的用户名和密码。

  1. 打开Git Bash（在开始菜单中搜索Git Bash）
  2. 输入以下命令生成SSH密钥（将your.email@example.com替换为您的GitHub邮箱）：
    3. ssh-keygen -t ed25519 -C "your.email@example.com"
  3. 按Enter键接受默认的密钥保存位置
  4. 输入密码（可选）或直接按Enter键不设置密码
  5. 再次输入密码确认或按Enter键
  6. 复制SSH公钥到剪贴板：
    7. cat ~/.ssh/id_ed25519.pub | clip
  7. 登录GitHub，点击右上角的头像，选择"Settings"
  8. 在左侧菜单中选择"SSH and GPG keys"
  9. 点击"New SSH key"
  10. 在"Title"字段中输入一个描述性名称
  11. 在"Key"字段中粘贴您的SSH公钥
  12. 点击"Add SSH key"
  13. 验证SSH连接：
    14. ssh -T git@github.com

    如果是首次连接，会提示您确认主机指纹，输入"yes"后按Enter键

    如果配置成功，您将看到成功的欢迎消息

第6步：使用Git将文件上传到GitHub

安装并配置好Git后，您可以使用以下步骤将您的乡村人才培训平台页面上传到GitHub：

  1. 首先，在GitHub上创建一个新的仓库（如果还没有）
  2. 打开命令提示符（CMD）或PowerShell，导航到您的项目文件夹：
    3. cd c:\trae
  3. 初始化Git仓库：
    4. git init
  4. 添加远程仓库（将URL替换为您的GitHub仓库URL）：
    5. git remote add origin https://github.com/yourusername/rural-talent-training.git
  5. 添加所有文件：
    6. git add .
  6. 提交更改：
    7. git commit -m "添加乡村人才培训平台页面"
  7. 推送更改到GitHub：
    8. git push -u origin main

    如果使用HTTPS方式，系统会提示您输入GitHub的用户名和密码（或个人访问令牌）

常见问题与解决方案

问题1："git"不是内部或外部命令

解决方案：

  • 确保Git已正确安装
  • 检查环境变量PATH是否包含Git安装路径
  • 关闭并重新打开命令提示符或PowerShell
  • 如果仍然无法解决，可以尝试重启计算机

问题2：无法连接到GitHub

解决方案：

  • 检查网络连接
  • 如果使用防火墙或代理，请确保已正确配置
  • 尝试使用SSH方式而不是HTTPS

问题3：推送被拒绝

解决方案：

  • 如果远程仓库不是空的，可能需要先拉取最新的更改：git pull origin main
  • 如果出现合并冲突，需要解决冲突后再提交

提示：安装完成后，您可以使用我们之前创建的上传脚本（如git_upload.py）来简化上传过程。
