# GitHub 代码上传使用说明

## 问题说明

您之前运行 `upload_to_github.bat` 时遇到的乱码问题，主要是由于文件编码格式不正确导致的。Windows 批处理文件需要使用 ANSI 或 UTF-8（无 BOM）编码才能正常显示中文。

## 解决方案

我们已经创建了两个新的批处理文件，它们使用了正确的编码格式：

1. `simple_upload_new.bat` - 主要的上传脚本
2. `test_batch_format.bat` - 用于测试批处理文件格式是否正确

## 使用步骤

### 步骤 1：先测试批处理文件格式

1. 打开文件资源管理器（按 Win+E 键）
2. 导航到 `C:\trae` 文件夹
3. 找到并双击 `test_batch_format.bat` 文件
4. 如果窗口中中文显示正常，没有乱码，说明批处理文件格式正确

### 步骤 2：使用新的上传脚本

1. 在 `C:\trae` 文件夹中，找到并双击 `simple_upload_new.bat` 文件
2. 按照窗口中的提示进行操作
3. 脚本会自动完成以下操作：
   - 检查 Git 是否安装
   - 初始化 Git 仓库（如果尚未初始化）
   - 设置 GitHub 远程仓库（使用 SSH 格式）
   - 拉取远程内容（如果有）
   - 添加所有文件
   - 提交更改
   - 推送到 GitHub

## 故障排除

如果运行 `simple_upload_new.bat` 时仍然遇到问题，请检查以下几点：

1. **Git 安装**：确保已正确安装 Git 并添加到系统环境变量
2. **SSH 密钥**：确保已在 GitHub 上配置了正确的 SSH 密钥
3. **网络连接**：确保网络连接正常
4. **仓库权限**：确保您对 GitHub 仓库有访问权限

如果问题仍然存在，请查看错误提示并尝试解决。脚本中包含了详细的错误信息，帮助您定位问题。
