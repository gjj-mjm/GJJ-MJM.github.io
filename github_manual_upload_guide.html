GitHub仓库手动上传指南

注意：本指南提供了不使用Git命令行工具，直接通过GitHub网站界面手动上传文件的方法。这种方法适合文件数量不多、不需要频繁更新的情况。

第1步：创建GitHub账号（如果还没有）

  1. 访问 GitHub官网
  2. 点击右上角的"Sign up"按钮
  3. 按照提示创建账号（输入用户名、邮箱和密码）
  4. 完成邮箱验证

第2步：创建新的GitHub仓库

登录GitHub后，按照以下步骤创建新仓库：

  1. 点击右上角的"+"图标
  2. 从下拉菜单中选择"New repository"
  3. 在"Repository name"字段中输入仓库名称（例如：rural-talent-training）
  4. （可选）在"Description"字段中添加仓库描述
  5. 选择仓库可见性：公开（Public）或私有（Private）
  6. （可选）勾选"Initialize this repository with a README"选项
  7. 点击"Create repository"按钮
创建仓库界面示意图

第3步：手动上传文件

仓库创建成功后，可以按照以下步骤上传文件：

方法一：直接上传文件

  1. 在仓库主页，点击"Upload files"按钮
  2. 将文件从本地文件夹拖拽到指定区域，或者点击"choose your files"选择文件
  3. 选择需要上传的HTML文件（如 index.html, contact_us.html, modern_agriculture.html 等）
  4. 上传完成后，在页面底部的"Commit changes"部分：
    • 在"Add file via upload"输入框中修改提交信息（例如："上传乡村人才培训平台页面"）
    • 选择提交到的分支（默认为main）
  5. 点击"Commit changes"按钮完成上传
上传文件界面示意图

方法二：通过Create new file功能

如果文件数量较少，也可以直接在GitHub上创建新文件并粘贴内容：

  1. 在仓库主页，点击"Create new file"按钮
  2. 在文件名输入框中输入文件名（例如：index.html）
  3. 在编辑区域粘贴文件内容
  4. 在页面底部输入提交信息
  5. 点击"Commit new file"按钮

第4步：上传多个HTML文件

由于GitHub限制单次上传文件数量，您可能需要分多次上传。以下是推荐的上传顺序：

  1. 首先上传主页：index.html
  2. 然后上传主要页面：contact_us.html
  3. 接着上传课程详情页面（可分批上传）：
    • modern_agriculture.html
    • traditional_crafts.html
    • rural-tourism-detail.html
    • 其他课程页面...
  4. 最后上传辅助文件，如：.gitignore

提示：GitHub允许单次上传最多100个文件，每个文件大小限制为25MB。

第5步：设置GitHub Pages（可选）

如果您想让您的网站可以通过GitHub Pages公开访问，请按照以下步骤操作：

  1. 在仓库页面，点击顶部的"Settings"选项卡
  2. 在左侧菜单中，点击"Pages"
  3. 在"Source"部分，点击下拉菜单，选择分支（通常为main或master）
  4. 选择文件夹（通常为"/(root)"）
  5. 点击"Save"按钮
  6. 等待几分钟，GitHub会生成网站链接
  7. 链接生成后，会显示在"GitHub Pages"部分（通常格式为：https://username.github.io/repository-name/）

提示：启用GitHub Pages后，任何人都可以通过生成的链接访问您的网站。

第6步：管理已上传的文件

上传文件后，您可以：

  • 查看文件：点击文件名即可查看内容
  • 编辑文件：点击文件页面右上角的铅笔图标进行编辑
  • 删除文件：点击文件页面右上角的垃圾桶图标进行删除
  • 下载文件：点击文件页面右上角的"Raw"按钮，然后右键保存

常见问题与解决方案

问题1：上传的HTML页面无法正确显示样式

解决方案：

  • 确保HTML文件中的CSS链接路径正确
  • 如果使用CDN引入CSS/JS库，检查链接是否有效
  • 如果使用相对路径，确保文件结构正确

问题2：无法上传大文件

解决方案：

  • GitHub限制单个文件大小为25MB
  • 对于超过限制的文件，可以使用Git LFS（Large File Storage）
  • 或者将大文件压缩后再上传

问题3：GitHub Pages未生成链接

解决方案：

  • 等待几分钟，链接生成可能需要时间
  • 确保仓库中有可作为主页的HTML文件（如index.html）
  • 检查分支和文件夹设置是否正确

额外提示

  • 文件命名：保持文件名一致，避免使用特殊字符和空格
  • 文件组织：对于多个相关页面，可以考虑创建文件夹进行分类
  • 备份：定期备份本地文件，不要仅依赖GitHub存储
  • 版本管理：每次重大更改时添加有意义的提交信息，便于追踪更改历史
  • 协作：如果多人协作，可以使用分支和Pull Request功能

重要提示：请勿在GitHub上上传敏感信息，如密码、API密钥等。如果您需要使用这些信息，请考虑使用环境变量或配置文件，并将其添加到.gitignore文件中。
